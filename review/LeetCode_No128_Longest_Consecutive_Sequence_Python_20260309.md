# LeetCode No.128 Longest Consecutive Sequence Code Review

## 알고리즘 코드 리뷰: LeetCode No.128 Longest Consecutive Sequence

### 1. 시간 복잡도 (Big-O 분석 및 최적화 가능성)

*   **`nums_set = set(nums)`**: 리스트를 세트로 변환하는 과정은 `N`개의 요소가 있을 때 평균적으로 O(N)의 시간이 소요됩니다. 해시 충돌이 최악의 경우 O(N^2)까지 갈 수 있으나, 일반적인 정수 입력에서는 O(N)으로 간주합니다.
*   **`for num in nums_set:`**: 외부 반복문은 `nums_set`에 있는 고유한 숫자 `N_unique`개만큼 반복됩니다. 최악의 경우 `N_unique`는 `N`과 같습니다.
*   **`if (num - 1) not in nums_set:`**: 세트에서의 `in` 연산은 평균적으로 O(1)입니다. 이 조건문은 `num`이 연속 시퀀스의 시작점인지 효율적으로 판단합니다. 이 부분이 전체 시간 복잡도를 O(N)으로 유지하는 핵심입니다. 만약 `num - 1`이 세트에 있다면, 즉 `num`이 이미 다른 시퀀스의 중간에 포함된 숫자라면, 해당 `num`에 대해서는 내부 `while` 루프가 실행되지 않습니다.
*   **`while current in nums_set:`**: 내부 `while` 루프는 현재 시퀀스의 길이에 비례하여 반복됩니다.
*   **종합 분석**: `for` 루프와 `while` 루프를 합쳐서 보면, `nums_set`의 각 숫자는 `while` 루프의 `current` 변수에 의해 **최대 한 번만** 방문됩니다. 예를 들어 `[1, 2, 3, 4, 5]` 시퀀스에서 `num=1`일 때 `while` 루프가 `1, 2, 3, 4, 5`를 모두 탐색합니다. 이후 `num=2, 3, 4, 5`로 `for` 루프가 돌 때는 `if (num - 1) not in nums_set` 조건이 거짓이 되므로 `while` 루프가 실행되지 않습니다.
*   **최종 시간 복잡도**: 따라서 모든 `set` 연산(삽입, 조회)이 평균 O(1)임을 고려할 때, 전체 시간 복잡도는 **O(N)** (N은 `nums` 리스트의 요소 수)이 됩니다. 이는 이 문제를 해결할 수 있는 가장 최적의 평균 시간 복도입니다.
*   **최적화 가능성**: 현재 풀이는 이미 시간 복도 측면에서 최적화되어 있습니다. O(N)보다 더 빠른 시간 복잡도를 달성하기는 어렵습니다.

### 2. 공간 복잡도

*   **`nums_set = set(nums)`**: `nums` 리스트의 모든 고유한 요소를 저장하기 위한 세트입니다. 최악의 경우 (모든 요소가 고유할 때) `N`개의 요소를 저장하므로 O(N)의 공간이 필요합니다.
*   **최종 공간 복잡도**: **O(N)** (N은 `nums` 리스트의 요소 수)입니다.

### 3. 보편적인 "정석" 풀이 방법

*   제공된 풀이 방식은 "Longest Consecutive Sequence" 문제에 대한 가장 보편적이고 **정석적인(canonical)** 접근 방식입니다. 해시 테이블(Python의 `set`이 내부적으로 해시 테이블로 구현됨)의 O(1) 평균 시간 조회를 활용하여 O(N) 시간 복잡도로 문제를 해결합니다.
*   다른 접근 방식으로는 배열을 먼저 정렬(O(N log N))한 후 선형 탐색하는 방법이 있지만, 이는 이 세트 기반 접근보다 느립니다. 동적 프로그래밍(DP)이나 그래프 알고리즘은 이 문제에 직접적으로 적합하거나 효율적인 해결책이 아닙니다.

*   **적절성**: 매우 적절하고 효율적인 풀이입니다. 문제의 핵심을 정확히 파악하고 최적의 자료구조를 활용했습니다.
*   **엣지 케이스 처리**:
    *   **빈 리스트 (`[]`)**: `nums_set`이 비어있어 `for` 루프가 실행되지 않고 `result`는 초기값 `0`을 반환합니다. **정확합니다.**
    *   **단일 요소 리스트 (`[1]`)**: `nums_set`은 `{1}`이 되고, `num=1`에 대해 `(0 not in nums_set)`이 참이 되어 `while` 루프가 `1`을 확인하고 `length=1`을 반환합니다. **정확합니다.**

    *   **음수 포함 리스트 (`[-1, 0, 1]`)**: `set`에 음수도 저장되며, `num - 1` 연산도 문제없이 처리됩니다. **정확합니다.**
