# LeetCode No.20 Valid Parentheses - Code Review

## 1. 시간 복잡도 (Big-O Analysis)

**시간 복잡도: O(n)**

- 단순 반복문을 통해 문자열을 한 번만 순회하므로, 시간 복잡도는 O(n)입니다.
- 각 문자에 대해 스택의 마지막 요소를 확인하고, pop/append 연산은 평균 O(1)입니다.
- 이는 문제에 대한 최적의 시간 복잡도로, 모든 문자를 한 번씩 확인해야 하므로 이 이상 최적화할 수 없습니다.

## 2. 공간 복잡도

**공간 복잡도: O(n)**

- 최악의 경우(모든 문자가 여는 괄호인 경우), 스택에 모든 문자를 저장해야 하므로 공간 복잡도는 O(n)입니다.
- 이 또한 최적의 공간 복잡도이며, 입력 문자열의 길이에 따라 선형적으로 증가합니다.

## 3. 보편적인 "정석" 풀이 방법

**적절한 접근법: 스택 사용**

- 이 문제는 classic한 괄호 매칭 문제로, **스택(Stack)** 자료구조를 사용하는 것이 정석적인 접근법입니다.
- 여는 괄호는 스택에 푸시하고, 닫는 괄호가 나올 때 스택의 마지막 요소를 비교하여 매칭이 되면 팝합니다.
- 이 풀이는 문제 유형에 맞는 **스택 기반의 괄호 매칭 알고리즘**으로, DP나 그래프와 같은 다른 접근법이 필요하지 않습니다.

## 4. 현재 풀이의 적절성 및 개선 가능 부분

### ✅ 장점:
- 문제를 정확하게 해결합니다.
- 코드의 논리가 직관적이고 간결합니다.

### ⚠️ 개선점:
1. **엣지 케이스 고려 부족**:  
   - `stack.length === 0` 일 때, `stack[stack.length - 1]`에 접근할 경우 배열이 비어 있는 상태에서 인덱스를 참조하므로 **런타임 에러** 발생 가능성이 있습니다.
   - 예: `s = ")"` 일 때, 스택이 비어 있는 상태에서 `stack[stack.length - 1]` 접근 시 오류

2. **코드 중복**:  
   - `if (parenthese === ")")`, `"}"`, `"]"` 각각에 대해 거의 동일한 로직이 반복됩니다.

3. **문자 비교 방식**:  
   - `==` 대신에 `===` 사용을 권장합니다. TypeScript에서는 타입 안정성과 일관성을 위해 `===`이 권장됩니다.

## 5. TypeScript의 언어적 특성(Idiomatic Code) 준수 여부

### ✅ 본질적으로 TypeScript를 잘 활용하고 있습니다:
- `const` 키워드를 사용하여 불변성을 유지
- 타입 명시(`string`, `boolean`)
- for...of 구문을 사용하여 반복

### ⚠️ 개선 가능 사항:
- `new Array()` 대신에 `[]` 리터럴 사용: 더 간결하고 TypeScript 친화적
- `stack[stack.length - 1]` 대신에 스택의 top 요소를 더 명확하게 다루는 방식 고려
- `if` 문을 **map 또는 switch-case**로 리팩토링하여 중복 제거 가능

## ✅ 최종 개선된 코드 예시:

```typescript
function isValid(s: string): boolean {
    const stack: string[] = [];
    
    for (const char of s) {
        if (char === ")" || char === "}" || char === "]") {
            if (
                stack.length > 0 &&
                ((char === ")" && stack[stack.length - 1] === "(") ||
                 (char === "}" && stack[stack.length - 1] === "{") ||
                 (char === "]" && stack[stack.length - 1] === "["))
            ) {
                stack.pop();
            } else {
                return false;
            }
        } else {
            stack.push(char);
        }
    }

    return stack.length === 0;
}
```

### ✅ 개선 사항 요약:
- `stack` 배열에 타입 명시
- 예외 상황(스택이 비었을 때) 처리 추가 (런타임 에러 방지)
- `if` 문 중복을 하나로 통합하여 가독성 향상
- `===` 사용으로 타입 비교 명확화
